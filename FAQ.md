# Frequently Asked Questions (FAQ)

A centralized reference for common questions about the AI‑Ops Framework, its structure, usage, and integration patterns.

---

## 1. What is the purpose of the AI‑Ops Framework?

The framework provides a structured, modular operating system for teams.  
It defines workflows, templates, system models, and documentation that enable clarity, alignment, and AI‑assisted execution.

---

## 2. How do I get started?

1. Read the `README.md`  
2. Review the `MODULE_REGISTRY.md`  
3. Explore workflows, templates, and system models  
4. Follow the Operating Playbook for weekly execution  

---

## 3. How do AI agents use this framework?

AI agents load:

- System models (logic)  
- Templates (structure)  
- Workflows (process)  
- Prompts (execution)  

This ensures consistent, structured outputs.

See: `docs/ai-agent-integration.md`

---

## 4. What is the difference between workflows, templates, and systems?

- **Workflows** define *process*  
- **Templates** define *structure*  
- **Systems** define *logic*  

Together, they form the core of the operating model.

---

## 5. How do I add a new workflow, template, or system?

1. Create the file in the correct folder  
2. Follow naming conventions  
3. Add it to `MODULE_REGISTRY.md`  
4. Ensure dependencies are documented in `docs/dependency-map.md`  

---

## 6. How do I know which components to load for a task?

Use this order:

1. Systems  
2. Templates  
3. Workflows  
4. Prompts  

This ensures the AI understands logic → structure → process → execution.
