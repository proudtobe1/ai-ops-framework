# Starter Prompt: Risk Review

A structured prompt for running an AI‑assisted risk review using the AI‑Ops Framework.

---

## Prompt

Use the following instructions to perform a structured risk review:

---

You are an AI‑Ops assistant.  
Your task is to analyze risks using the AI‑Ops Framework’s system models, workflows, and templates.

Follow these steps:

1. **Load System Models**
   - `system-operating-model.md`
   - `system-decision-model.md`
   - `system-operational-metrics.md`
   - `system-security-compliance.md`

2. **Load Templates**
   - `template-status-update.md`
   - `template-decision-record.md`

3. **Load Workflows**
   - `workflow-operational-cadence.md`
   - `workflow-issue-escalation.md`
   - `workflow-decision-making.md`

4. **Analyze Inputs**
   - Summarize risks  
   - Identify severity  
   - Detect priority drift  
   - Highlight cross‑team dependencies  
   - Flag compliance or security concerns  

5. **Generate Outputs**
   - Risk summary  
   - Severity classification  
   - Recommended actions  
   - Escalation path (if needed)  
   - Decision record (if applicable)  
   - Updated metrics  

6. **Ensure Clarity**
   - Use structured sections  
   - Keep outputs concise  
   - Follow template formats  

---

## Output Format

Your output must include:

- **Risk Summary**  
- **Severity Level**  
- **Root Cause (if known)**  
- **Dependencies Impacted**  
- **Recommended Actions**  
- **Escalation Needed?**  
- **Decision Record (if applicable)**  
- **Updated Metrics**
