# Template: Dependency Review

A structured template used by AI agents to evaluate dependencies, assess readiness, identify blockers, classify risks, update metrics, and determine whether escalation is required.

---

## 1. Summary
Provide a concise overview of dependency health, readiness, and key findings.

---

## 2. Context / Inputs
List all inputs used for the dependency review, including:
- Team updates  
- Roadmap priorities  
- Sprint goals  
- Upstream dependencies  
- Downstream dependencies  
- Risks  
- Metrics  
- Missing information  

---

## 3. Dependency Inventory
List all active dependencies with brief descriptions.

### **Dependency 1**
- Description  
- Upstream or downstream  
- Owner (if provided)  

### **Dependency 2**
- Description  
- Upstream or downstream  
- Owner (if provided)  

(Repeat as needed)

---

## 4. Readiness Assessment
Evaluate readiness for each dependency:
- Upstream readiness  
- Downstream readiness  
- Delivery confidence  
- Timeline health  

---

## 5. Dependency Risk Classification
Classify dependency‑related risks using the standard severity scale:
- **High** — Requires immediate escalation  
- **Medium** — Requires mitigation  
- **Low** — Monitor only  

Include rationale for each classification.

---

## 6. Impact Analysis
Evaluate dependency impact across:
- Strategic impact  
- Execution impact  
- Risk impact  
- Metrics impact  
- Communication impact  

---

## 7. Alignment Impact
Evaluate how dependencies affect alignment across:
- Strategic alignment  
- Execution alignment  
- Dependency alignment  
- Risk alignment  
- Metrics alignment  
- Communication alignment  

Provide an updated **alignment score** using the 0.0–1.0 scale.

---

## 8. Mitigation Status
For each dependency, document:
- Current mitigation steps  
- Owner (if provided)  
- Progress status  
- Blockers  
- Required support  

---

## 9. Escalation Evaluation
Determine whether escalation is required based on:
- Severity  
- Impact  
- Drift indicators  
- Compliance concerns  
- Timeline risk  

If escalation is required, specify:
- Escalation type  
- Escalation urgency  
- Triggering conditions  

---

## 10. Metrics Updates
Update relevant metrics:
- Dependency health metrics  
- Dependency risk metrics  
- Alignment score metrics  
- Cadence metrics  

---

## 11. Recommended Actions
Provide clear, actionable next steps to resolve dependency issues and restore alignment.

---

## 12. Communication Summary
Produce a concise summary suitable for stakeholders.

---

## 13. Compliance Notes
Document any compliance‑relevant observations.

---

## 14. Audit Log
Include:
- Timestamp  
- Actor  
- Inputs used  
- Dependencies evaluated  
- Severity classifications  
- Escalation decisions  
- Compliance notes
