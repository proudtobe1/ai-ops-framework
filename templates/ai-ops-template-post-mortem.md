# AI Agent Post‑Mortem Template

A comprehensive, audit‑ready incident analysis template used to document failures, drift events, misalignments, outages, or unexpected agent behavior within the AI‑Ops Framework.

---

## 1. Incident Summary

**Incident Title:**  
**Incident Type:** (Failure / Drift / Misalignment / Outage / Compliance Issue / Other)  
**Severity:** Low / Medium / High / Critical  
**Timestamp:**  
**Status:** Resolved / Ongoing  

**Summary:**  
(2–4 sentences describing the incident)

---

## 2. Timeline

- **T‑0:** Incident begins  
- **T‑1:** Detection  
- **T‑2:** Initial response  
- **T‑3:** Escalation (if applicable)  
- **T‑4:** Mitigation  
- **T‑5:** Resolution  
- **T‑6:** Post‑incident actions  

Include timestamps for each event.

---

## 3. Context & Inputs

- User request  
- System models involved  
- Workflows involved  
- Templates involved  
- Dependencies  
- Constraints  
- Missing information  

---

## 4. Incident Details

### **A. What Happened**
- Description of the incident  
- Expected behavior vs. actual behavior  
- Systems or components affected  

### **B. Detection**
- How the issue was detected  
- Signals or indicators  
- Who or what identified the issue  

### **C. Impact**
- Operational impact  
- Alignment impact  
- Dependency impact  
- Compliance impact  
- Audit impact  
- User impact  

---

## 5. Root‑Cause Analysis (RCA Summary)

Summarize the RCA findings:

- Primary root cause  
- Contributing factors  
- Failure mode classification  
- Evidence  

(Full RCA is stored separately.)

---

## 6. Mitigation Actions

- Immediate mitigation  
- Long‑term mitigation  
- Owner  
- Timeline  
- Required resources  

---

## 7. Preventive Actions

- Safeguards added  
- Workflow updates  
- Template updates  
- Model updates  
- Monitoring updates  
- Training updates  

---

## 8. Metrics Impact

- Risk metrics  
- Alignment metrics  
- Cadence metrics  
- Audit metrics  
- Operational metrics  

Include before/after values if available.

## 9. Compliance Notes

- Compliance rules triggered  
- Classification  
- Redaction  
- Escalation required: Yes / No

## 10. Audit Notes

- Traceability  
- Reproducibility  
- Logging status

## 11. Lessons Learned

- What worked

## 12. Stakeholder Summary

A concise summary suitable for cross‑team or leadership communication.
